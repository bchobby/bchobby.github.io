二、Amazon Q 震撼来袭

在距离开发者最近的应用层，亚马逊云科技和微软、Google 之间的竞争，正在从彼时的云计算领域，延展到当下的生成式 AI 领域。这一次在前有 ChatGPT、Bard 的夹击下，亚马逊云科技于本次大会上推出了一个名为 Amazon Q 的 AI 助手与其“兵戎相见”。

和前二者有所不同的是，Amazon Q 是由亚马逊云科技 17 年来积累的知识和经验训练而成，目标更为精准，主要是面向企业级用户的工具。用户可以通过亚马逊云科技管理控制台、文档页面、IDE、Slack 或其他第三方对话应用程序的聊天界面访问 Amazon Q。

2.1 什么是Amazon Q？

Amazon Q（供 AWS Builder 使用）是一款生成式人工智能 (AI) 支持的对话助理，可以帮助您理解、构建、扩展和操作 AWS 应用程序，并回答有关 AWS 的问题，包括但不限于有关支持、架构、最佳实践和文档。Amazon Q 不断更新其功能和能力，确保您的问题获得最与上下文相关且可操作的答案，包括在需要时将您与人工联系的能力。

在 IDE 中使用时，Amazon Q 还提供软件开发帮助，包括代码解释、代码生成以及调试和优化等代码改进。

Amazon Q 由Amazon Bedrock提供支持，这是一项完全托管的服务，可通过 API 提供基础模型 (FM)，并通过高质量的 AWS 内容进行了增强，可为您提供更完整、可操作且可参考的答案，以加速您在 AWS 上的构建。

2.2 Amazon Q功能介绍

  • 聊天 – Amazon Q 可以用英语回答有关 AWS 的自然语言问题，包括有关 AWS 服务选择、AWS CLI 使用、文档和最佳实践的问题。Amazon Q 以信息摘要或分步说明进行响应，并包含对其信息源的引用。
  • 记忆 – Amazon Q 使用您的对话上下文来告知您对话期间的未来响应。
  • 代码改进和建议 – 在 IDE 中，Amazon Q 可以回答有关软件开发的问题、改进代码并生成新代码。
  • 故障排除和支持 – Amazon Q 可以帮助您了解 AWS 控制台中的错误，并提供实时支持代理来解决您的 AWS 问题。
  • 客户反馈 – Amazon Q 使用您通过反馈表单提交的信息和反馈来提供支持或帮助解决技术问题。

如果照片有声音，此刻你将听到全场开发者的欢呼 —— Amazon Q Code Transformation 可以在极短的时间内完成 Java 等平台的语言版本迁移和更新，提升安全性和性能水平，加速从 Windows 迁移到 Linux 的迁移（Coming Soon）等。

三、Amazon Q测评体验

3.1 使用 Amazon Q 排查控制台错误

在 AWS 管理控制台中，Amazon Q 可以排查您在使用 AWS 服务时收到的错误，例如权限不足、配置不正确、超出服务限制等。此功能适用于在 AWS 控制台中使用 Amazon EC2、Amazon S3、AWS Lambda 和 Amazon ECS 时出现的错误。

举例：

启动一台云实例，“故意”将该路由表中的子网取消关联，进行连接测试，查看是否能够连接上去。最后，再使用Amazon Q进行检测，排查故障原因所在。

显示报错信息如下：

NO_ROUTE_TO_DESTINATION: 路由表 rtb-04c1cef73eec90cc3 中没有到 igw-0ba9932fb8577bd6b 的适用路由。

重新导航至VPC，进行查看有关路由表相关配置，重新关联上子网。

此时，云实例可以正常连接访问了。

3.2 与 AWS 支持人员聊天

可以使用 Amazon Q 创建支持案例，并从 AWS 管理控制台（包括 AWS 支持中心控制台）中的任何位置联系 AWS Support。Amazon Q 使用您的对话上下文自动代表您起草支持案例。最终确定案例详细信息并创建案例后，Amazon Q 可以通过您选择的联系方式将您转接给支持代理，包括在同一界面中进行实时聊天。

Amazon Q官方文档快速导航：https://docs.aws.amazon.com/amazonq/latest/aws-builder-use-ug/what-is.html
