案例二：

```py

测试 https://weibo.cn/pub/，抓取个人详细资料：https://weibo.cn/xxxx/info

重点：

直接访问个人资料页，页面编码格式是 UTF-8 但是还是报编码格式错误，

是因为没有进入到个人信息页面，而是跳转到了登录页面，登录页面不是 UTF-8 所以报错，也就是校验信息没带全，算没登录

引用 urllib

import urllib.request

请求头

headers = { # 校验 ua ，防爬方式的一种 'user-agent': '', # 主要就是少了这一行，cookie 携带着登录信息，如果有登录之后的 cookie，那么就能通过携带 cookie 进入任何页面，防爬方式的一种 'cookie': '', # 用于判断当前路径是不是由上一个路径进来的，如果不是也会报错，一般情况是做图片的防盗链，防爬方式的一种，这种方式属于可选防爬方式不是每个网站都校验这个 'referer': 'https://weibo.cn/' }

请求地址

url = 'https://weibo.cn/xxx/info'

请求对象

request = urllib.request.Request(url=url, headers=headers)

模拟浏览器向服务器发起请求

response = urllib.request.urlopen(request)

如果报错编码格式错误，可以通过输出 read() 内容查看网页编码，但是只适用个别网页

print(response.read()) #

获取响应数据

content = response.read().decode('utf-8')

print(content)

将页面数据保存，方便确认当前在哪(打开保存的文件会发现是 登录页面)

with open('weibo.html', 'w', encoding='utf-8') as f: f.write(content) ```
