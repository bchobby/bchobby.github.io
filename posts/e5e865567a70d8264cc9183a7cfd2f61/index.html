VRRP小实验

一、实验内容

实验背景：

作为公司的网络管理员，当前的网络中有两个用户，PC1、PC2模拟为公司用户，R1使用环回口模拟Internet服务器。此时网络中有两个网关，为实现冗余选择使用VRRP协议。实验拓扑如下图所示：

二、实验目的

1.了解网络负载均衡的功能和作用。
2.理解VRRP协议的工作原理。
3.掌握三层交换环境单组VRRP的配置方法，及其优化配置。

三、实验方案设计

本次实验采用一台路由器，两台具备三层路由功能的交换机，两台终端设备（PC），网络中划分了三个VLAN,分别为VLAN 1、VLNA 2、VLAN 3。
1.在SW1上创建VLAN 1、VLNA 2、并且配置如上图拓扑所示IP地址，然后将G0/0/10接口配置为trunk模式，允许vlan 1 通过，将G0/0/1接口配置为access模式，划分vlan 2。
2.在SW2上创建VLAN 1、VLNA 3、并且配置如上图拓扑所示IP地址，然后将G0/0/10接口配置为trunk模式，允许vlan 1 通过，将G0/0/1接口配置为access模式，划分vlan 3。
3.PC1连接SW1的G0/0/2口，并且将SW1的G0/0/2口配置为access模式，划分vlan 1 。
4.PC1连接SW2的G0/0/2口，并且将SW2的G0/0/2口配置为access模式，划分vlan 1 。
5.在SW1上进入vlanif 1,配置VRRP为Master设备端，vrid 为1，虚拟网关地址为10.0.123.254 ，设置vrrp优先级105（默认值为100，越大越优），开启抢占模式（默认开启），设置抢占延迟时间为10秒，开启上行链路追踪功能。
6.在SW2上进入vlanif 1,配置VRRP为Backup设备端，vrid 为1，虚拟网关地址为10.0.123.254 。
7.在路由器AR1上配置接口IP如上图所示，并且配置loopback0来模拟外网网段。然后开启OSPF路由协议，在区域0里，宣告两个接口IP地址和一个loopback0接口地址。
8.SW1也开启OSPF路由协议，在区域0里，宣告vlan 1 和 vlan3 网段。
9.SW2也开启OSPF路由协议，在区域0里，宣告vlan 1 和 vlan2 网段。
10.PC1和PC2的网关地址均为10.0.123.254。

四、实验配置命令

SW1:

#
Vlan batch 2       //创建vlan2，
#
interface Vlanif1
 ip address 10.0.123.2 255.255.255.0    
 vrrp vrid 1 virtual-ip 10.0.123.254     //虚拟网关地址
 vrrp vrid 1 priority 105      //修改VRRP优先级，使得SW1变为Master设备
 vrrp vrid 1 preempt-mode timer delay 10    //配置抢占延迟时间，使得业务流量不中断
 vrrp vrid 1 track interface GigabitEthernet0/0/1 reduced 20  //配置追踪上行链路，快速感知链路故障。
#
interface Vlanif2
 ip address 10.0.11.1 255.255.255.0
#
interface GigabitEthernet0/0/1
 port link-type access
 port default vlan 2              //接口划为vlan2。
#
interface GigabitEthernet0/0/2
 port link-type access
 port default vlan 1           //接口划为vlan1，华为接口默认PVID为vlan 1。
#
interface GigabitEthernet0/0/10
 port link-type trunk
 port trunk allow-pass vlan 1  //允许vlan1通过该链路，华为设备默认允许vlan1通过。
#
ospf 1                         //进入OSPF视图
 silent-interface Vlanif1   //设置静默接口，禁止接口发送OSPF报文，防止路由震荡。
 area 0.0.0.0                //进入区域0
  network 10.0.11.1 0.0.0.0      //宣告接口IP地址
  network 10.0.123.0 0.0.0.255   //宣告内网IP网段

SW2

#
Vlan batch 3       //创建vlan3，
#
interface Vlanif1
 ip address 10.0.123.3 255.255.255.0    
 vrrp vrid 1 virtual-ip 10.0.123.254     //虚拟网关地址
#
interface Vlanif3
 ip address 10.0.12.1 255.255.255.0
#
interface GigabitEthernet0/0/1
 port link-type access
 port default vlan 3              //接口划为vlan3。
#
interface GigabitEthernet0/0/2
 port link-type access
 port default vlan 1           //接口划为vlan1，华为接口默认PVID为vlan 1。
#
interface GigabitEthernet0/0/10
 port link-type trunk
 port trunk allow-pass vlan 1  //允许vlan1通过该链路，华为设备默认允许vlan1通过。
#
ospf 1                         //进入OSPF视图
 silent-interface Vlanif1   //设置静默接口，禁止接口发送OSPF报文，防止路由震荡。
 area 0.0.0.0                //进入区域0
  network 10.0.12.1 0.0.0.0      //宣告接口IP地址
  network 10.0.123.0 0.0.0.255   //宣告内网IP网段

验证结果：

1.正常情况下，SW1的vlan 1 为VRRP的Master设备端，SW2的vlan 1 为VRRP的Backup设备端。

2.网络正常情况下，PC1访问R1的loopback0接口走的路径是 PC1-SW1-AR1

3.当SW1设备发送故障时候，如上行链路断开后，PC1访问R1的loopback0接口走的路径是PC1-SW2-AR1，此时SW2将变成vlan1 的VRRP 的Master端。

4.由于配置VRRP的抢占延迟和追踪上行链路，保证VRRP的业务流量不中断，提高网络的稳定性，现在在PC1一直Ping AR1的loopback0接口地址，然后断开SW1的上行接口，查看验证结果。

五、实验小结

本次实验主要了解VRRP的配置参数的优化，认识什么是负责均衡模式，理解抢占延迟时间设置的原因，上行链路追踪的原因，有兴趣的同学还可以做VRRP联动BFD会话，进行上行链路的故障检测，收敛级别处于毫秒级别，收敛更快。本次实验相对简单，因为只做了当个vrrp组的主备，这个是初始的环境有助于我们理解工作原理，以小见大的提高实验兴趣。现网内一般都是VRRP+MSTP组合使用，接入层设备需要做冗余链路，汇聚层设备之间做聚合链路，多个VRRP组进行负载均衡，提高网络的可靠性，实现网关的冗余，注意VRRP的Master端与MSTP的根桥要对应一致，以免出现次优路径。汇聚层设备与出口网关设备设置冗余的拓扑环境，出口网关上采用双线双宿主的连接方式，提高公网出口的高可靠性。
