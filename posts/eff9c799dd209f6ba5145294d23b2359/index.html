组合优化：是关于寻找元素的子集

组合优化问题的示例：
1.背包
2.图中的最小生成树
3.图中的最短路径
4.最小顶点或集合覆盖

背包问题是组合优化中的一个问题：给定一组项目，每个项目都有一个权重和一个值，确定要包含在集合中的每个项目的数量，以便总权重小于或等于给定的限制，并且总值尽可能大。
当前解~x在每次递归调用时都会求值（重量和价值）。那么所有变量x1到xz都被认为是固定的。

枚举：枚举背包的所有有效解决方案
这个问题对应于一个集合的所有子集的枚举物品（不包括背包中不适合的子集）。
解向量和优化函数：针对每个电流解向量~x=（x1，…，xn）我们保持：当前总值vcurr和当前总重量wcurr
全局变量：
最佳解向量~xbest和关联的最佳总价值vmax
方法：分而治之

这个方法会枚举所有的可能结合的方式,追踪所有的数据,停止当得到最佳结果时.这是一个递归过程.

执行：算法首先调用Enum（0，0，0，~0）。
第一个是number z of already assigned variables in ~x;第二个是total value
vcurr;第三个是total weight wcurr;第四个是 current solution-vector ~x.
这样受限的搜索空间通过考虑是否包括z+1项，以这种方式创建的所有子问题都是递归解决的。
运行时：每个n项和每个递归调用的工作量是恒定的（不包括递归）。因此，总运行时间为O（2n).

边界：对于我们计算的每个子问题的局部上界U和局部下界L

取消：子问题U≤ L（L对应于a全局下限）不再需要考虑！
全局下限(Global lower bound)是目前最好的解决办法

界限：任何有效解的值都是一个下限。上界将使用所谓的对偶函数单独计算
启发法。

背包：改进的下界

排序：项目按相对值vi/wi进行排序.
下限：一个人贪婪地在背包里装满物品变量还没有固定，直到没有更多的项目可根据容量进行包装

我们之前已经看到了一个简单的上限。
可能的改进：
变量尚未确定的所有项目都是按给定顺序考虑。
最主要的改进方式是用vi/wi 的值所生成的顺序来选择符合容量的物品.
如果不能改进一个分支,那么就忽视掉那个分支,看其他的.

最初的最好的值默认为负无穷或者布尔值来表示找到了解决办法或没有.把一整个问题分解成小问题,然后找到 这个办法的上下界限(lower and upper bound)如果上限在被改进,会着重注意这一分支;如果没有,则忽视掉这一分支.

最小化顶点覆盖
我们将通过以下方式表示部分解决方案和当前子实例：
已经有顶点覆盖中的一组顶点C，以及从原始图中得到的子图G,移除C。终点将放在剩下的顶点们.
Enum( G, C, N ):
if E(G) = ∅ and |C| < |Cmin|
Cmin ← C
if E(G) 6= ∅
choose a non-isolated vertex v of G that is not in N
Enum(G − {v}, C ∪ {v}, N )
Enum(G, C, N ∪ {v} )
非正式的：按照一定的顺序遍历所有顶点并且顶点分支取决于它是否位于顶点覆盖中。并且选择过程中,顺序是不会改变的.但是选择特殊的顶点决定用或者不用它,这会逐渐地影响到决策.
最后,所有的边会被移除,这个图象会变为空.这意味着,已经成功地构造顶点覆盖.现在的顶点数将会少于最小的数量.这样做的目的就是得到最小的顶点覆盖值.如果还有边,那么就选择一个不独立的顶点.因为如果是独立的,它没有边,在定义上没法跟其他任一顶点连接,无法组成顶点覆盖.
C是顶点覆盖中的顶点。
N是我们选择不包含在顶点覆盖中的顶点。

如果一个顶点属于顶点覆盖那么直接去除所有的与它相连接的边;如果顶点不在顶点覆盖中，则其所有邻居们一定是。
这将导致每个顶点v有两个分支：
将v放入顶点覆盖或将v的所有邻域放入顶点覆盖中。
在优化中,不仅仅是去除边还去除用边连接的顶点来简化.

开始的时候是一个空集没有任何边,
紧接着G ← G − {u, v}.最后移除完就得到了下限(lower bound)
G ← G − {u, v}的含义是:挑选一个点H 并且加入H 和其它的连接,再去除掉所有被连接到的

lower bound 选的是两个顶点见的线段,去除的是与那条线段连接的部分的线段,所有的顶点依旧在原本的位置不变;而upper bound 首先选择有最多连接线的顶点去除该顶点的所有连线,当还剩下两个分段的线段后,选择任一一条线段的其中一个顶点消除那条线段,最终就只剩下一条单独的线段(选择留下剩下两个中有着较大数值的顶点的线段)
